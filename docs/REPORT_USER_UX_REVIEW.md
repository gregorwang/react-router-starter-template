# 报告：用户视角的体验缺口与优化建议

## 首次使用体验
- 进入即跳转对话列表，缺少新手引导与配置检查。
- 未配置密钥/归档/AI 时只报错，不指引如何修复。

## 操作效率
- 项目新建使用浏览器 `prompt`，缺少校验与描述字段。
- 对话列表缺少搜索、重命名、删除、置顶、标签。
- 无对话“撤回/重试/重新生成”入口。

## 反馈与可控性
- 异常通过 `alert()` 提示，打断流程。
- 流式中断后没有“继续/重试/复制部分内容”。

## 认知负担
- 模型参数分散在顶部栏，缺少“当前会话设置概览”。
- 无统一设置页，用户不知道“我到底配了哪些模型/密钥”。

## 建议的体验优化
1. 首次使用向导
- 检测关键环境变量是否配置，提示下一步操作。

2. 操作面板
- 将“新建项目、重命名、删除、搜索”集中到统一对话管理面板。

3. 失败友好
- Toast + 可重试按钮，取代 `alert()`。

4. 对话能力增强
- 复制/分享/导出、单条消息标记、对话收藏。

5. 设置中心
- 模型默认值、预算/限速、主题、数据保留。

## 预期价值
- 降低首次使用流失率
- 提升日常操作效率
- 增强可控性与信任感
